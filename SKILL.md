# 类人搜索（Human Find）

## 核心理念

不要指望一次搜对关键词。像人类一样找工具：**先找到种子，再顺藤摸瓜。**

人类找工具的路径：随便搜 → 找到半相关的 → 看它的 awesome list/引用链 → 去社区看安利帖 → 找到真正好的。

本技能模拟这个过程，分三步走。

---

## 第一步：种子搜索

**目标：找到任何相关的种子项目，不求精准，只求有关。**

用以下关键词分别在 skills.sh 和 GitHub 上搜索，**每个关键词都要搜，全部展示结果**：

### 关键词生成规则

对用户的原始需求，生成以下四组关键词：

1. **用户原话直译**：把用户说的话翻译成搜索词
2. **需求领域的通用术语**：这个领域的人常用什么词？
3. **近义/关联词**：同义词、上位词、下位词
4. **中文/英文各一版**：用户用中文就先搜中文，再搜英文；反之亦然

### 搜索工具

#### skills.sh 注册表

```bash
npx skills find [关键词]
```

对每组关键词都执行一次，记录所有返回的结果（名称、安装量）。

#### GitHub 搜索

使用 WebSearch：

- `[关键词] site:github.com skill`
- `[关键词] github agent skill`

### 透明度要求

**必须向用户展示：**
- 每个搜了什么关键词
- 每个关键词返回了哪些结果（名称、star 数/安装量、一句话描述）
- 哪些结果看起来相关，哪些不相关

**禁止一句话概括**，比如不能写"找到了一些相关结果"，必须列出具体内容。

---

## 第二步：扩散搜索（最关键的一步）

**目标：从种子出发，顺着引用链找到更好的项目。**

对第一步找到的每个种子项目：

1. **打开它的 GitHub README**，找以下内容：
   - "Awesome"列表、"Related Projects"、"参考项目"、"类似工具"
   - 它引用了哪些其他项目
   - 它的 issues 里有没有人提到替代品

2. **如果种子项目本身是一个 awesome list**（收录列表）：
   - 直接扫描整个列表，找最符合用户需求的项目
   - 按 star 数排序，取前 5 个

3. **如果种子项目有相关项目链接**：
   - 逐个打开，检查是否更符合用户需求
   - 继续看这些项目的 README，是否有更深的引用

4. **在 skills.sh 上查看种子项目的来源**：
   - 同一个来源（owner）下可能还有其他相关 skill

### 为什么这一步最重要？

因为 awesome list 和引用链里藏着社区已经筛选过的高质量项目。你不需要自己判断哪个好，社区已经帮你判断了。很多高质量项目不会出现在搜索结果的前几条，但它们会被收录在 awesome list 或被其他项目引用——顺着引用链走，往往一步到位。

---

## 第三步：社区搜索

**目标：看别人已经整理好的推荐帖。**

使用 WebSearch 搜索：

- `[用户需求关键词] 推荐 2025`
- `[用户需求关键词] skill 推荐`
- `[用户需求关键词] site:bilibili.com`
- `[用户需求关键词] site:xiaohongshu.com`
- `[用户需求关键词] best tools reddit`

社区帖子里往往已经有人整理好了对比和推荐，比自己搜效率高得多。

---

## 结果整理

### 排序规则

所有找到的结果按 GitHub star 数降序排列。star 数相差 10 倍以上的必须调整顺序。

### 每个结果必须包含

| 字段 | 说明 |
|------|------|
| 名称 | skill 名称 |
| 来源渠道 | 在第几步、用什么关键词找到的 |
| GitHub star 数 | 核心质量标尺 |
| 功能描述 | 一句话说明它做什么 |
| 安装命令 | `npx skills add ...` 或 `git clone ...` |
| 链接 | GitHub 或 skills.sh 链接 |

### 找不到时的报告

如果三步都没找到，明确告诉用户：
- 每一步搜了什么关键词，返回了什么结果
- 为什么这些结果不符合需求
- 主动问用户：你有没有听说过相关的项目？

## 自检清单

在给出最终结果前，检查：

- [ ] 是否展示了每个关键词的搜索结果？
- [ ] 是否打开了种子项目的 README 看了引用链？
- [ ] 是否搜索了社区推荐帖？
- [ ] 结果是否按 star 数降序排列？
- [ ] 每个结果是否标注了来源渠道？
