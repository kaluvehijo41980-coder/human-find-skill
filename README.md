# Human-Find Skill 🔍

类人搜索工具（Human Find）—— 采用“种子扩散”策略，像人类一样在庞大的开源社区中为您挖掘最优质的 AI 技能和工具。

## 💡 核心理念

永远不要指望一次就搜对完美的关键词。像人类一样找工具的法则：**先找到种子，再顺藤摸瓜。**

人类找工具的真实路径往往是：`随便搜` → `找到半相关的` → `顺着看它的 awesome list 或是引用链` → `去社区看推荐帖` → `找到真正最顶级的平替`。本技能（Skill）完全模拟并自动化了这个过程。

## 🚀 智能工作流

### 1️⃣ 第一步：种子搜索
将您的需求自动裂变为多维度的关键词（原话直译、行业术语、近义词、中英双语），在 `skills.sh` 注册表和 GitHub 上进行广泛撒网。此步不求精准，只求找到任何相关的“种子项目”。

### 2️⃣ 第二步：扩散搜索（核心灵魂）
从种子出发，顺藤摸瓜：
- 深入分析种子项目的 `README`（寻找 Awesome 列表、Related Projects、参考项目等）。
- 追踪种子项目引用的底层工具或其他项目。
- 挖掘同作者仓库里的其他宝藏。
>*为什么这一步最重要？因为我们不需要自己去判断哪个项目好，社区真实的“引用链”和收录列表已经帮我们做好了优胜劣汰。*

### 3️⃣ 第三步：社区经验吸收
在 Reddit、Bilibili、小红书等内容社区搜索相关的“推荐盘点帖”。站在前人的肩膀上，直接吸收别人耗时整理好的对比和安利。

## 📥 如何使用

本工具是一个规范的 AI Agent Skill，可以直接集成到您的智能体工作流中。

**触发情景**：
当您向 Agent 提出类似 **"有没有做 X 功能的 skill？"** 或 **"帮我在 GitHub 找个 X 工具"** 时，Agent 将自动激活本技能。

**输出交付**：
经过三步深度挖掘后，Agent 会严格按客观热度（GitHub Star 数或安装量）为您输出一份结构化的调研报告，包含：
- **项目名称**与**功能描述**
- **来源渠道**（透明展示是通过哪一步搜到的）
- **客观热度**（Star / 安装量）
- **一键安装命令**与**直达链接**
